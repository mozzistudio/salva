### 4.2 Content Integration Without Breaking Minimalist Design

#### The "Iceberg" Principle

```

â”œâ”€â”€ Technical specs

```

#### Placement Strategies

**Above the Fold (Minimal, High-Impact)**
```yaml
Content Type: Headlines, trust badges, primary CTA
Word Count: 50-100 words
SEO Role: H1 keyword placement, meta description match
Design: Clean, spacious, Beats-inspired
```

**First Scroll (Progressive Disclosure)**
```yaml
Content Type: Service categories, key benefits
Word Count: 150-300 words
SEO Role: H2 keywords, semantic coverage
Design: Cards or icons with short descriptions
```

**Mid-Page (Expandable Depth)**
```yaml
Content Type: Detailed explanations, FAQ, process
Word Count: 400-800 words (in accordions)
SEO Role: Long-tail keywords, featured snippet opportunities
Design: Collapsed by default, clean when expanded
```

**Bottom of Page (SEO Foundation)**
```yaml
Content Type: Full service descriptions, location details
Word Count: 300-500 words
SEO Role: Internal links, local SEO signals, comprehensive coverage
Design: Can be slightly denser, users expect it here
```

## 7. SEO Constraints for Future Design

### 7.1 Non-Negotiable SEO Elements

These must be present regardless of design preferences:

#### Critical Elements

| Element | Requirement | Reason |
|---------|-------------|--------|
| **H1 visible above fold** | Must be real `<h1>`, not image | Crawlability + accessibility |
| **Text-based navigation** | Links must be crawlable `<a href>` | Googlebot needs text |
| **Mobile-first layout** | Content parity mobile/desktop | Mobile-first indexing |
| **Page speed < 3s** | LCP under 2.5s target | Core Web Vitals ranking factor |
| **Semantic HTML** | Proper heading hierarchy | Machine readability |
| **Alt text on images** | Descriptive, keyword-aware | Image SEO + accessibility |
| **Structured data** | LocalBusiness, FAQ, Service schemas | Rich results eligibility |
| **Internal links** | Text links between key pages | Crawl paths + PageRank |
| **Meta tags** | Unique title + description per page | SERP appearance |
| **Canonical URLs** | Self-referencing canonicals | Duplicate prevention |

#### Content Requirements

| Requirement | Minimum | Notes |
|-------------|---------|-------|
| **Visible text** | 300+ words above fold | Can't be all images |
| **Total page text** | 800+ words (Inicio), 1000+ words (Empresas) | Including expandables |
| **FAQ section** | 5+ questions | Schema markup opportunity |
| **Location info** | Full address, phone, hours | Local SEO signals |

---

### 7.2 Design Decisions That Harm SEO (Avoid)

#### Layout Anti-Patterns

| Bad Decision | SEO Impact | Alternative |
|--------------|------------|-------------|
| **Full-screen video hero with no text** | No H1 for crawlers | Video + text overlay |
| **Infinite scroll** | Content not indexed | Paginated or load-more |
| **Text as images** | Not crawlable | Real text with CSS styling |
| **JavaScript-rendered content only** | May not be indexed | SSR or static HTML |
| **Hamburger menu on desktop** | Hides important links | Visible navigation |
| **Lazy-load above-the-fold content** | LCP penalty | Eager load hero |

#### Technical Anti-Patterns

| Bad Decision | SEO Impact | Alternative |
|--------------|------------|-------------|
| **Blocking CSS/JS in robots.txt** | Can't render page | Allow all resources |
| **No HTTPS** | Ranking penalty + trust | SSL certificate required |
| **Duplicate content across pages** | Cannibalization | Unique content per page |
| **Orphan pages** | Not crawled/indexed | Internal link to every page |
| **4xx/5xx errors** | Wasted crawl budget | Fix all broken links |
| **Slow server response** | Crawl issues | TTFB < 200ms |

#### Content Anti-Patterns

| Bad Decision | SEO Impact | Alternative |
|--------------|------------|-------------|
| **Thin pages (< 300 words)** | Won't rank | Add expandable content |
| **Keyword stuffing** | Penalty risk | Natural language |
| **Hidden text (white on white)** | Spam penalty | Visible or properly hidden |
| **Same meta title/description** | Poor differentiation | Unique per page |
| **No location mentions** | Poor local ranking | Panama in key positions |
| **Generic alt text ("image1.jpg")** | Missed opportunity | Descriptive alt text |

---

### 7.3 Design-SEO Compromise Guidelines

#### The "Minimalist + SEO" Balance

| Design Want | SEO Need | Compromise |
|-------------|----------|------------|
| Clean hero, minimal text | H1 must be visible | Short, punchy H1 (5-8 words) |
| Beautiful images | Alt text required | Write good alt text (invisible to users) |
| Single-page feel | Multiple pages for keywords | Clear internal linking |
| Animation on scroll | Content must be in DOM | Animate visible, not loaded |
| White space | Enough text for ranking | Expandable sections for depth |
| No cluttered footer | Links needed | Clean, organized footer nav |

#### The "Beats-Inspired" Checklist

```yaml
Beats does this well (keep):
  - Strong visual hierarchy
  - Headline-first communication
  - Progressive disclosure
  - White space confidence

Beats can afford to ignore (Salva cannot):
  - SEO text depth (they have brand searches)
  - Local signals (they're global)
  - Detailed FAQs (brand awareness carries them)
  - Long-tail keyword coverage (branded searches dominate)
```
